## REAL BUGS surfaced (both fixed in this commit)

### 1. `/bonus` threw `UNAUTHORIZED` for the entire unauthenticated visitor cohort

The landing page CTA `All N →` (in the bonus-preview section) links to `/bonus`. That page was inside the `(app)` route group and called `requireSession()` — which throws `new Error("UNAUTHORIZED")` when no session is present. Every unauthenticated click on `All N →` produced a Next.js error page, not a marketing surface. The first run-3 capture caught the bare-stack error screen (Next.js error overlay + `src/lib/auth/session.ts:53` highlighted) instead of the bonus library.

This is a **conversion-killing bug** that landing-only runs could not have surfaced — capturing only `/` never traverses the link. The funnel-walk capture caught it on the second page.

**Fix shipped:** `(app)/bonus/page.tsx` and `(app)/bonus/[slug]/page.tsx` now use the soft `getSession()` (returns null when unauthenticated) instead of `requireSession()`. Free / pro gating keys off the session: no session → free; session + tier=pro → pro; everything else → free. Unauth visitors see the same `FREE_PREVIEW_COUNT = 3` first cards open, the rest locked with a "Sign up to read" CTA pointing at `/signup?from=bonus-<slug>` (the trial path) instead of `/pricing?from=…` (upgrade path that doesn't apply to a non-user). The page also now shows a hero CTA pair (Start free / See pricing) when unauthenticated, instead of the dashboard-orientated "← Dashboard" back link.

### 2. Bonus library polluted with placeholder titles from inaccessible scrapes

Three bonus entries (`05-ig-post-DWX1Jk1k-UK.md`, `06-ig-post-DXqo18qlCRm.md`, `12-fb-server-hardening.md`) have raw permalink hashes as their H1 titles — they were scraped from private Instagram posts / login-gated Facebook shares and never received human-readable titles. They were rendering on `/bonus` as cards titled literally "Instagram Post DWX1Jk1k-UK," "Instagram Post DXqo18qlCRm," and "Facebook Share: …". **All three personas independently caught these** (Sarah: "single biggest polish gap on the site"; Priya: "auto-titling didn't run on the scraped content"; Dave: "kills credibility on a CISO skim and burns the trust the rest of the funnel builds").

**Fix shipped:** `app/src/lib/content/bonus.ts` now applies a placeholder-title filter at parse time:
```ts
const isPlaceholderTitle = (title: string): boolean =>
  /^(Instagram Post\s+[A-Za-z0-9_-]+|Facebook Share[: ]|Inaccessible|Login required)/i.test(
    title.trim(),
  );
```
The three files remain in `docs/content/` for future content-completion work (per the user's WhatsApp ingestion pipeline memory) but are hidden from public listings until they have real titles. The user-facing bonus count drops from 16 → 13.

## False positives from vision misreads — methodology rule reinforced

Run 3 reproduced the documented vision-on-small-tokens unreliability **twice more**:

| Persona | Read as | Actually rendered | Source line |
|---|---|---|---|
| Priya | `"33967 engines"` | `"OSINT engines"` | `pricing/page.tsx:59` |
| Priya | `"Python IRs"` | `"Python libs"` | `pricing/page.tsx:59` |
| Priya | `"Day 17 crypto"` | `"Day 13 crypto"` | `pricing/page.tsx:76` |
| Priya | `"mirrors real v12"` | `"mirrors real v13"` | `pricing/page.tsx:79` |
| Sarah | `"code: 13c1ac"` footer artifact | (none — does not exist in source) | grep -r returns 0 hits |
| Marcus (run 2) | `"a explained quiz questions"` | `"4 explained quiz questions"` | `page.tsx:78` |
| Sarah (run 2) | `"x explained quiz questions"` | `"4 explained quiz questions"` | `page.tsx:78` |
| Sarah (run 1) | `"65% of CEH v13 pass mark"` | `"80%"` (now `"70%"`) | `page.tsx:404` |
| Alex (run 1) | `"60% of CEH v13 pass mark"` | `"80%"` (now `"70%"`) | `page.tsx:404` |
| Priya (run 1) | `"80% of CEH v13 pass mark"` ✓ | `"80%"` ✓ | `page.tsx:404` |

**Pattern:** small grey monospace text — 2-digit numbers, 3-5 character tokens like "libs" / "v13" — produces unreliable vision extraction. Larger display text (the hero headline, CTAs, section headers) reads cleanly across all runs. Anything quoted from those zones can be trusted; numbers and short mono tokens **must** be cross-referenced against rendered HTML or source before being promoted into a "copy bug" finding.

This is now strong enough evidence to bake into the harness as an explicit rule (recorded in HANDOFF + qa README).
