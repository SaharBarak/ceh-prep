## Directory Purposes

**`app/src/app/`:**
- Purpose: Next.js App Router pages and layouts
- Contains: `.tsx` files (pages, layouts, error boundaries), client components, route groups
- Key files:
  - `layout.tsx`: Root layout (metadata, fonts, html/body wrapper)
  - `page.tsx`: Home / splash page
  - `(auth)/layout.tsx`, `(auth)/login/`, `(auth)/signup/`: Unauthenticated routes
  - `(app)/layout.tsx`: Protected shell (redirects to /login if no session)
  - `(app)/dashboard/page.tsx`: Progress overview + day cards
  - `(app)/course/[day]/`: Dynamic course player per day
  - `(app)/exam/`: Exam simulator (125-question sheet)

**`app/src/lib/`:**
- Purpose: Reusable domain logic, infrastructure, data access
- Structure: Organized by concern (auth, db, validation, content, actions, dto)
- Not a dumping ground: each subdir has a single responsibility

**`app/src/lib/auth/`:**
- Purpose: Authentication and security services
- Contains:
  - `session.ts`: iron-session integration; `getSession()`, `requireSession()` helpers
  - `password.ts`: Argon2id hashing (OWASP 2024 tuning); verify function
  - `rate-limit.ts`: Sliding-window rate limiter; in-process LRU with swappable Redis interface
  - `hibp.ts`: k-anonymity check for pwned passwords
- Pattern: Each function is pure or has predictable side effects (network, crypto)

**`app/src/lib/db/`:**
- Purpose: MongoDB connection, schema definition, type inference
- Contains:
  - `mongo.ts`: Global connection pool (prevents hot-reload connection spam in dev)
  - `models/user.ts`: User schema (email unique, passwordHash, tier, lockout counters)
  - `models/progress.ts`: Quiz progress schema (userId + day unique compound index; always query by userId)
  - `models/audit.ts`: Audit event schema (event, outcome, ip, ua, meta, userId)
- Pattern: Export Mongoose Model and InferredType for each entity; no DAO classes (Mongoose methods used directly)

**`app/src/lib/validation/`:**
- Purpose: Zod schemas for all external input (FormData, query params, JSON)
- Contains:
  - `schemas.ts`: SignupSchema, LoginSchema, SaveAnswerSchema
- Pattern: Define reusable sub-schemas (email, password) to avoid duplication

**`app/src/lib/content/`:**
- Purpose: Static curriculum data and types
- Contains:
  - `types.ts`: Domain, Question, Day, Concept, Exercise types (readonly)
  - `days.ts`: 14-day curriculum entries (hardcoded)
  - `index.ts`: Exports DAYS array, getDay(n), isFreeDay(n) helpers
- Pattern: Data is immutable; lookups are O(1) or O(N) with .find() on small array

**`app/src/lib/dto/`:**
- Purpose: Explicit mapping from internal docs to client-safe shapes
- Contains: UserPublic type; toPublicUser() converter
- Pattern: Allow-list approach; never leak passwordHash, failedLoginCount, lockedUntil
- Used by: Server actions before returning user data to client

**`app/src/lib/actions/`:**
- Purpose: Server Actions — the command layer
- Contains:
  - `auth.ts`: signup, login, logout (form-based, FormData parsing)
  - `progress.ts`: saveAnswer, getUserProgress, getDayAnswers (JSON RPC-style)
- Pattern: "use server" directive at file top; return typed results or throw; no direct response.json() (Next.js handles serialization)

## Key File Locations

**Entry Points:**
- Root layout: `app/src/app/layout.tsx` — metadata, fonts, html/body
- App root: `app/src/app/page.tsx` — splash page (public)
- Middleware: `app/src/middleware.ts` — header injection only; no auth

**Configuration:**
- Environment: `app/src/lib/env.ts` — Zod schema validation at boot
- TypeScript: `app/tsconfig.json` — strict mode; path alias @/* → src/*
- Next.js: `app/next.config.ts` — security headers (CSP, HSTS, X-Frame-Options)

**Core Logic:**
- Auth services: `app/src/lib/auth/` — session, password, rate-limit, hibp
- Models: `app/src/lib/db/models/` — user, progress, audit
- Server actions: `app/src/lib/actions/` — signup, login, saveAnswer, progress
- Schemas: `app/src/lib/validation/schemas.ts` — Zod input validation

**Testing:** (not yet implemented; see TESTING.md when ready)
- Test files: co-located with source or under `app/src/__tests__/` (TBD)
- Test data: fixtures in `app/src/__tests__/fixtures/` (TBD)

## Naming Conventions

**Files:**
- `kebab-case` for files: `auth.ts`, `rate-limit.ts`, `hibp.ts`, `login-form.tsx`, `course-player.tsx`
- `[brackets]` for dynamic segments: `[day]` in route
- `(parentheses)` for route groups: `(auth)`, `(app)`
- `+page.tsx`, `+layout.tsx` for route definitions

**Directories:**
- `kebab-case` for directory names: `lib/auth/`, `lib/db/`, `lib/actions/`, `app/course/`
- Route groups in parentheses: `(auth)`, `(app)`

**TypeScript:**
- `PascalCase` for components: `LoginForm`, `CoursePlayer`, `ExamRunner`
- `camelCase` for functions: `getDay()`, `saveAnswer()`, `isPasswordPwned()`, `toPublicUser()`
- `SCREAMING_SNAKE_CASE` for constants: `DAYS`, `PARAMS` (in password.ts), `CSP`, `securityHeaders` (in next.config.ts)
- `snake_case` or PascalCase for types: Result<T, E>, SessionData, ActionErrorCode, UserDoc

**Variables:**
- `camelCase` for variables and parameters
- Single-letter vars allowed in tight loops: `for (const q of quiz)`, `const [k, v] of map`
- Descriptive names for functions: `isPasswordPwned()` not `checkPw()`, `getSession()` not `sess()`

## Where to Add New Code

**New Feature (e.g., password reset):**
- Domain logic: `app/src/lib/auth/password-reset.ts` (new file)
- Schema: Add ResetSchema to `app/src/lib/validation/schemas.ts`
- Model: Extend UserDoc with resetTokenHash, resetTokenExpiry; add index
- Server action: `app/src/lib/actions/auth.ts` — add passwordReset() function
- Routes: Create `app/src/app/(auth)/reset-password/` directory with page.tsx + form component
- Test: `app/src/__tests__/auth.test.ts` (co-located or in __tests__ dir)

**New Route (e.g., account settings):**
- If public: Create `app/src/app/settings/page.tsx`
- If protected: Create `app/src/app/(app)/settings/page.tsx`
- If needs separate form: Add `app/src/app/(app)/settings/settings-form.tsx` marked "use client"
- If needs server action: Add function to existing or new action file

**New Component:**
- Client components: `app/src/app/[route]/[component-name].tsx` marked "use client"
- Server components: Default, no "use client" directive
- Reusable: Consider moving to `app/src/lib/components/` (future, not yet implemented)

**New Model / Database Schema:**
- Add to `app/src/lib/db/models/[entity].ts`
- Export Model<DocType> and InferSchemaType
- Add indexes for query patterns (userId, email, day, created timestamps)
- Enforce unique constraints at schema level when needed

**New Utility / Helper:**
- Shared across actions: `app/src/lib/[concern]/[helper].ts` (e.g., `lib/auth/mfa.ts`)
- One-off in action: Keep inline unless reusable
- Pure function: Add to `app/src/lib/` subdirectory without side effects

## Special Directories

**`app/src/lib/content/`:**
- Purpose: Immutable curriculum data
- Generated: No (hand-written or imported from external source)
- Committed: Yes
- Modification: Update `days.ts` with new day entries; schema changes in `types.ts`; re-export in `index.ts`

**`app/public/`:**
- Purpose: Static assets (favicon, social preview images)
- Generated: No
- Committed: Yes
- Served: Next.js serves directly from /public root

**`app/.next/`:**
- Purpose: Build output and cache
- Generated: Yes (build time)
- Committed: No (.gitignore)
- Do not edit: Regenerated on build

**`app/node_modules/`:**
- Purpose: Installed dependencies
- Generated: Yes (npm install)
- Committed: No (.gitignore)
- Update: npm install / npm update; lock file committed

**`free/`:**
- Purpose: Static HTML landing (no build, no Node.js)
- Generated: No
- Committed: Yes
- Deployment: Copy `index.html` directly to CDN or static host

## Import Path Aliases

**`@/*` → `app/src/*`:**
- Use `@/lib/auth/` not `../../../lib/auth/`
- Use `@/app/` not `./` (from components)
- Defined in `app/tsconfig.json` → "paths": { "@/*": ["./src/*"] }
